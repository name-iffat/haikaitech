# Product Marketing Context

*Last updated: May 29, 2026*

## Product Overview
**One-liner:** Freelance software engineer building web apps, dashboards, and interactive systems.
**What it does:** Full-stack development services from React frontends to .NET backends, Unity visual tools, and production systems.
**Product category:** Freelance Software Engineering / Web Development
**Product type:** Service (agency-of-one)
**Business model:** Custom per-feature / per-project pricing

## Target Audience
**Target companies:** Small-to-mid businesses, startups, and agencies in Malaysia needing web app development
**Decision-makers:** Founders, CTOs, product managers, agency owners
**Primary use case:** Building custom web apps, dashboards, and visual tools
**Jobs to be done:**
- "I need someone to build my web app from scratch"
- "I need to add interactive dashboards to my existing product"
- "I need a freelancer who can handle both frontend and backend"
**Use cases:**
- Full-stack web applications (React + .NET)
- Data dashboards and internal tools
- Interactive/visual systems (Unity, prototyping)
- Landing pages and marketing sites

## Problems & Pain Points
**Core problem:** Clients struggle to find a single developer who can handle both polished frontends and solid backends without needing multiple specialists.
**Why alternatives fall short:**
- Agencies are expensive and slow
- Individual freelancers often specialize in only frontend or backend
- Junior devs lack production experience
**Emotional tension:** Uncertainty about whether a freelancer can deliver quality work on time, fear of hiring someone who can't communicate technically.

## Competitive Landscape
**Direct:** Local Malaysian freelancers and small agencies
**Secondary:** Upwork / Freelancer.com generalists
**Indirect:** Hiring in-house junior devs

## Differentiation
**Key differentiators:**
- Full-stack capability (frontend + backend + interactive systems)
- Visual-first mindset (cares about how things look AND feel)
- Experience across React, .NET, Unity, and data tools
**Why that's better:** Clients get one person who can own the entire product — from database to UI to interactive experiences.
