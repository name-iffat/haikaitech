## Entry: Debug — Broken Project Images (Root Cause)
**Time**: 02:07 MYT
**Theme**: Identified and fixed `src="[object Object]"` in SSR output

### Main Topics
- Discovered homepage project cards rendered `<img src="[object Object]">` in SSR
- Traced to Astro's asset import behavior: `import img from './src/assets/foo.webp'` returns `ImageMetadata {src, width, height, format}`, not a plain URL string
- `constants.tsx` stored the full metadata object in `Project.image` (typed `string`)
- `ProjectCard.tsx` and `HeroSection.tsx` both used the object directly as `<img src={...}>`
- Same issue affected `HeroSection.tsx` with its `avatarImg` import
- Build was passing because TypeScript didn't catch the type mismatch (inference from Vite import)

### Key Insights & Learnings
- Astro image imports under `src/assets/` always return `ImageMetadata` objects, even in non-Astro `.tsx` files imported by Astro pages
- The `Project.image: string` type in `types.ts` is a type error — it should be `string` but the assignment accepts `ImageMetadata` because of inference
- SSR React renders `[object Object]` when a non-string object is used as `src` attribute
- The production build output (content-hashed URLs like `/assets/foo.hash.webp`) and dev server paths (`/src/assets/foo.webp`) use different URL spaces — explaining why curl to raw paths worked but browser showed broken images

### What Went Well
- Diagnosis took one `dist/index.html` SSR output inspection
- Fix was simple: add `.src` to each image assignment (2 files, 9 sites total)
- Build passed cleanly after fix
- All 29 optimized image variants intact

### What Could Be Improved
- Inconsistent asset import types between `.astro` (`ImageMetadata`) and `.tsx` (URL string) is a footgun
- `types.ts` should use `string` for `Project.image` but there's no compile-time enforcement that the constant values are `.src` — only runtime behavior reveals the mismatch

### Looking Forward
- Image handling is now stable: React islands get URL strings, `.astro` pages get `<Image />` with full optimization
- Consider using `getImage()` in `index.astro` to pass optimized responsive URLs as props to React island cards
- Add type annotation on image imports as a guardrail for future contributors
